# Codex Handoff

## 引き継ぎテンプレート（入力）
- 目的
- 完了条件（受け入れ条件）
- 非対象（やらないこと）
- 制約（ブランチ、依存、期限、禁止操作）
- テスト範囲（必須コマンド）
- 期待出力（PR / 差分要約 / 調査メモ）

## 手順
1. 曖昧語を具体化する（「適切に」などを禁止）。
2. 完了条件を検証可能な文に直す。
3. 非対象を明記してスコープを固定する。
4. 失敗時の扱い（リトライ条件、エスカレーション条件）を先に決める。
5. 最終依頼文を1本化する。

## 出力フォーマット
- 実施内容サマリ
- 変更ファイル一覧
- テスト結果
- 未解決事項
- 次アクション

## 禁止
- 目的と完了条件がない依頼を渡す
- テスト範囲なしで実装を渡す
- スコープ未固定のまま着手する
