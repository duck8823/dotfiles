# Issue Triage

## 入力
- Issue 一覧
- マイルストーン情報
- 現在の制約（人員、期限、依存）

## 手順
1. 各 Issue の目的と完了条件を明確化する。
2. 優先度を分類する（P0/P1/P2 など）。
3. 依存関係とブロッカーを明示する。
4. 大きすぎる Issue は分割案を作る。
5. 今スプリント対象と次点を分ける。

## 出力フォーマット
- 優先順位リスト
- 各 Issue の受け入れ条件
- 依存関係マップ（簡易で可）
- 分割提案
- 未確定事項と確認先

## 禁止
- 受け入れ条件なしで優先度だけ決める
- 依存関係を無視した計画
- 実装規模の見積もりなしで過積載する
