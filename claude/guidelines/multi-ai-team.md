# Multi-AI チーム設計

## コンセプト

同じ役割を Claude / Codex / Gemini の3AIで多重化し、各AIの得意領域を活かした異なる視点でレビュー・計画・仕様策定を行う。

## 役割×AI マトリクス

| ロール | Claude (サブエージェント) | Codex (TOML) | Gemini (MD) |
|--------|------------------------|--------------|-------------|
| **Planner** | 依存分析・Wave構成 | フィジビリティ・リスク | 優先度・マイルストーン整合 |
| **Spec** | ファイルスコープ・完了条件 | テスト戦略・セキュリティ | 既存パターン・影響範囲 |
| **Architect** | 依存追跡・レイヤー違反 | 設計推論・責務分離 | 俯瞰・命名一貫性 |
| **Reviewer** | バグ・テスト漏れ | セキュリティ・エッジケース | パターン一貫性・diff外 |
| **QA** | 探索的テスト | — | — |
| **Designer** | デザインシステム準拠 | — | — |

QA/Designer は Claude のみ（実機操作・ファイル直読みが必要なため）。

## エージェント定義の配置

```
project/
├── .claude/agents/     # Claude サブエージェント（Sonnet）
├── .codex/agents/      # Codex エージェント（Opus, reasoning_effort 調整）
├── .gemini/agents/     # Gemini エージェント（gemini-2.5-pro）
└── AGENTS.md           # クロスツール共通コンテキスト
```

## 共通出力形式（全エージェント統一）

```json
{
  "source": "<ai>-<role>",
  "findings": [
    {"severity": "CRITICAL|HIGH", "file": "path:line", "issue": "...", "fix": "..."}
  ],
  "summary": "問題なし" or "N件検出"
}
```

## 統合判断ルール（Claude Opus メイン）

1. 各ロールの3AI結果を読み込み
2. 同じファイル:行に対する指摘を統合（2AI以上 → 高信頼）
3. 1AIのみの指摘 → ソースコード実読で誤検出フィルタ
4. CRITICAL は無条件採用
5. 統合結果を `gh pr comment` で投稿
6. CRITICAL あり → 修正 → 修正箇所のみ Claude reviewer で再レビュー

## 契約プラン

| サービス | プラン | 主な役割 |
|---|---|---|
| Claude | Max / Pro | 実装・最終レビュー・マージ判断・Computer Use |
| ChatGPT | Pro / Plus | Codex で多重レビュー・計画・仕様・テスト作成・技術調査 |
| Google AI | Pro / Free | Gemini CLI で多重レビュー・計画・仕様・壁打ち |

## Claude に集中させるタスク（制限消費の価値あり）
- コーディング（実装・リファクタリング・バグ修正）
- Computer Use（UI テスト・シミュレーター操作）
- 最終レビュー・統合判断
- PR 作成・コミット分割

## Codex に振るタスク
- 多重レビュー（セキュリティ + 設計推論の観点）
- 多重計画（フィジビリティ + リスク分析の観点）
- 多重仕様（テスト戦略 + エッジケースの観点）
- テスト作成・実行
- オーバーフロー実装（Claude 制限到達時）

## Gemini に振るタスク
- 多重レビュー（俯瞰一貫性の観点）
- 多重計画（優先度 + マイルストーン整合の観点）
- 多重仕様（既存パターン + 影響範囲の観点）
- 壁打ち（長コンテキストで全体俯瞰）

## Claude 制限到達時のフォールバック
1. 実装タスク → Codex に非同期で投げる
2. レビュー → Gemini + Codex で実施
3. 調査・対話 → ChatGPT で対応
