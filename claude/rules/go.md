# Go 共通ルール

- アーキテクチャ: レイヤードアーキテクチャ（Presentation → Application → Domain ← Infrastructure）。リポジトリインターフェースは Domain 層に定義し、依存方向は内向き
- エラーハンドリング: `xerrors.Errorf` でスタックトレース付きラップ。`fmt.Errorf` は使わない。エラーメッセージは日本語で記述（OSS プロジェクトでは英語を優先）
- ドメインモデル: ファクトリ関数は `New`（エンティティ生成）/ `Of`（値オブジェクト・同等の情報から）/ `From`（変換）。エンティティはポインターレシーバー、値オブジェクトは値レシーバー
- インフラストラクチャ: SQL は `go:embed` で外出し。datasource パターンで単一構造体がリポジトリとクエリサービスを同時実装
- Linter: `revive` + `wrapcheck` + `staticcheck` を有効化。`//nolint` 使用時は必ず理由コメント
- テスト: テーブル駆動サブテスト。`cmp.Diff` でアサーション。テストケース名は日本語で条件を表現（OSS プロジェクトでは英語を優先）
- 命名: `types` パッケージで値オブジェクト・DTO を統一管理。パッケージは小文字・区切りなし
- NULL 許容値は `Optional[T]` で扱い、ポインタは使わない

詳細: `conventions/go/` 配下の各規約ファイルを参照
